Hệ thống Đấu giá Trực tuyến (Auction System)

Bài tập lớn nhóm tại UET - VNU | Vai trò: Core Developer

Context (Bối cảnh)
Hệ thống yêu cầu xử lý các phiên đấu giá công khai real-time, cho phép hàng chục kết nối client tương tác đồng thời cùng lúc mà không xảy ra sai sót dữ liệu tài chính hoặc đồng bộ thời gian.
Task (Nhiệm vụ)
Thiết kế hạ tầng kết nối mạng Server chịu tải ổn định, giải quyết triệt để bài toán race condition khi có nhiều người dùng cùng nhấn nút nâng giá tại cùng một mili-giây, và cấu trúc hệ thống lưu trữ database tối ưu.
Action (Hành động kỹ thuật)
  • Triển khai kiến trúc **Multi-threaded Socket Server** trong Java để cấp phát luồng xử lý độc lập cho từng kết nối client.
  • Áp dụng các cơ chế khóa đồng bộ (**Synchronization / Concurrency Controls**) để quản lý tiến trình ghi dữ liệu dồn dập, loại bỏ hoàn toàn xung đột giá thầu.
  • Sử dụng **MySQL Workbench** để chuẩn hóa mô hình ERD, viết các câu lệnh và thủ tục truy vấn SQL tối ưu hóa tốc độ phản hồi real-time.
  • Ứng dụng các mẫu thiết kế hướng đối tượng (**Singleton, Factory, Decorator**) và nguyên lý **SOLID** để module hóa backend sạch sẽ, dễ bảo trì.
Result (Kết quả đạt được)
Hệ thống vận hành ổn định với độ trễ tối thiểu, đáp ứng hoàn hảo kịch bản kiểm thử giả lập tải lớn khi hàng chục client tương tác giá đồng thời mà dữ liệu lịch sử phòng đấu giá luôn chuẩn xác 100%.
